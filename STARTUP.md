STARTUP FILE

0. When starting this project, do the following
    - Turn this file into markdown
    - Include the following contents / instructions in this file.
        - Every  time that you are asked to edit code in this repo, if there is a significant change to design or something that would help you more efficiently (no more than 200 lines) get up to speed during the next session, include it here (CLAUDE.md)
        - Create a file called README.md with the PURPOSE of the application, REPO STRUCTURE to explain to an unfamiliar developer how the files are layed out and the importance of each one, USAGE to explain how to interact with the repository and deploy code and teardown instructions including backup capabilities, ARCHITECTURE section to lay out the infrastructure design. This README.md file should also be updated after each run when Purpose/Repo Structure/Usage/Architecture have been meaningfully changed
        - Create a file called TESTING.md that lays out a TODO list with all the items that should be tested (unit testing, performance testing, and any potential security testing we should implement) as well as documentation on all testing that has been put in place. When an item has been removed from the TODO list and updated in the documentation below. This file should also be updated after each run when a feature that could require testing has been added.
